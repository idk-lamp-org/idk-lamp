# idk-lamp — 日本語版ドキュメント

このリポジトリは、**idk-lamp** の日本語版ドキュメントをまとめたものです。

idk-lamp は、AI システムにおける **「非決定（棄権）」を可視化するためのデザインシグナル** です。  
ガバナンス、規制、リスク管理の代替ではなく、  
**責任境界を明示するための最小限の UI プリミティブ** として機能します。

---

## 📘 このリポジトリに含まれる文書

### 1. `idk-lamp_概念マッピング.md`
idk-lamp と主要なグローバル AI ガバナンスフレームワーク  
（NIST AI RMF、EU AI Act、ISO/IEC 42001 など）との  
**非規範的な概念対応**をまとめた文書です。

- idk-lamp がどのように既存の概念と接続するか  
- どこが一致し、どこが異なるか  
- idk-lamp が「準拠」ではなく「翻訳」である理由  

を明確にします。

### 2. `idk-lamp_なぜ今なのか.md`
idk-lamp が **なぜ今必要なのか** を説明する背景文書です。

- AI が実質的な意思決定者になりつつある現状  
- AI が自然には「わからない」と言わない構造  
- 過剰自信・キャリブレーション不良の未解決性  
- ガバナンスに UI プリミティブが欠けている問題  

といった構造的変化に対する応答として  
idk-lamp が存在する理由を示します。

---

## 🎯 idk-lamp の位置づけ

idk-lamp は **ガバナンスフレームワークではありません**。  
以下のことは行いません：

- リスク評価  
- ポリシー強制  
- 意思決定  
- 標準・規制への準拠主張  

idk-lamp が提供するのはただ一つ：

> **AI が「決めていない」瞬間を、明確に可視化すること。**

これにより、責任境界が曖昧にならず、  
沈黙や欠落が「自信」と誤解されることを防ぎます。

---

## 🧭 文書体系の意図

このリポジトリの文書は、次の3つを目的としています：

1. **誤解を防ぐための境界の明示**  
   - idk-lamp はガバナンスではなく UI シグナルである  
   - 準拠文書ではなく概念翻訳である  

2. **Origins（起源）の透明性の確保**  
   - なぜ作られたのか  
   - どの問題に応答しているのか  
   - 何を意図し、何を意図しないのか  

3. **将来の拡張に耐える構造**  
   - 新しいフレームワークが追加されても破綻しない  
   - 実装パターンが増えても文書の役割が変わらない
